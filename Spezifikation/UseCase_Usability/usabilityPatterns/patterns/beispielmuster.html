Problem

Das Problem beschreibt ein typisches Interaktionsproblem (im weitesten Sinne), das Benutzer bei der Arbeit mit einem interaktiven Softwaresystem haben.

Lösung

Löse das Beispielproblem!

Die Lösung beschreibt eine

  1. abstrakte und vor allem
  2. bewährte

funktionale Lösung für das Problem.

Beschreibungen können auch auf andere Usability Patterns verweisen, etwa so: Globales Undo

Nutzungskontext

Der Nutzungskontext beschreibt, in welchen Situationen das Usability Pattern angewendet werden sollte. Der Nutzungskontext kann sich auf verschiedene Dinge beziehen, z.B.:

  • Benutzerkreis
  • Arbeitsaufgaben
  • Systemumgebung
Begründung

Die Begründung beschreibt, weshalb das Usability Pattern aus ergonomischer Sicht sinnvoll ist.

Risiken, Nachteile, Kosten
